# 🐱 AI 챗봇 '비사' 트러블슈팅 및 개선 기록

이 문서는 블로그 AI 챗봇 시스템에서 발생한 주요 문제들과 그에 대한 해결 과정을 기록합니다.

## 1. Google Gemini API 지역 제한 (User Location Not Supported)

### ❌ 문제 상황
- **증상:** 챗봇에게 메시지를 보내면 `"냥... User location is not supported for the API use."`라는 에러 메시지가 출력됨.
- **원인:** Cloudflare Worker는 전 세계 데이터 센터에서 실행되는데, 특정 노드(예: 유럽/영국 등)에서 Google AI Studio API를 호출할 때 지역 제한이 걸림.

### ✅ 해결 방법
1.  **API 엔드포인트 변경:** 기존 `v1beta`에서 더 안정적인 `v1` 엔드포인트로 전환.
2.  **인사말 즉시 응답 로직:** "안녕", "ㅎㅇ" 등 단순 인사말은 API를 거치지 않고 Worker 내부에서 즉시 응답하도록 `isGreeting` 로직을 강화하여 API 호출 횟수를 줄이고 에러 가능성을 차단.
3.  **프록시 지원 설계:** 향후 동일 문제 재발 시 우회할 수 있도록 `GEMINI_BASE_URL` 환경 변수 지원 구조 검토.

## 2. 블로그 포스트 URL 경로 불일치 및 파일명 표준화

### ❌ 문제 상황
- **증상:** 챗봇이 제공하는 '바로가기' 링크를 클릭하면 404 에러가 발생함.
- **원인:** 포스트 파일명이 `26-02-11-title.md`와 같이 연도가 2자리(`YY`)로 작성됨. Jekyll은 이를 유효한 날짜로 인식하지 못해 날짜를 제목(Slug)의 일부로 포함시킴으로써 실제 경로와 불일치 발생.

### ✅ 해결 방법
1.  **파일명 일괄 변경:** 모든 포스트 파일명을 `YY-MM-DD-`에서 `20YY-MM-DD-` (4자리 연도) 표준 형식으로 일괄 변경하여 Jekyll 표준 준수.
2.  **스크립트 로직 보강:** `generate_problems_json.py`에서 날짜 접두사를 완벽히 제거하도록 정규표현식 로직을 강화하여 순수 슬러그 주소 생성.

## 3. Cloudflare Worker 데이터 캐싱 문제 (Persistent Old Data)

### ❌ 문제 상황
- **증상:** 로컬 데이터(`problems.json`)와 스크립트를 수정하여 GitHub에 푸시했음에도 불구하고, 챗봇이 계속해서 날짜 접두사가 포함된 예전의 잘못된 링크를 반환함.
- **원인:** Cloudflare Worker가 성능 최적화를 위해 GitHub의 JSON 데이터를 Edge 노드에서 강력하게 캐싱(Caching)하고 있었음. 일반적인 배포만으로는 캐시가 즉시 지워지지 않음.

### ✅ 해결 방법
- **캐시 버스팅(Cache Busting) 적용:** `fetch` 요청 시 URL 뒤에 실시간 타임스탬프 파라미터를 추가(`url?t=Date.now()`)하고, 헤더에 `Cache-Control: no-cache`를 설정하여 Cloudflare 서버가 무조건 GitHub에서 최신 데이터를 가져오도록 강제함.

## 4. UI 가독성 및 가시성 문제

### ❌ 문제 상황
- **증상:** 기존 휴대폰 모양의 좁은 UI로 인해 긴 텍스트와 문제 리스트가 한눈에 들어오지 않음.

### ✅ 해결 방법
1.  **카카오톡 테마 개편:** 가로폭을 최대 600px로 확장하고, 친숙한 카카오톡 스타일(배경색, 말풍선 디자인)을 적용하여 가독성 대폭 향상.
2.  **파일명 정규화:** `music-player.html`을 `chatbot.html`로 변경하여 프로젝트 정체성 확립.
3.  **반응형 레이아웃:** PC와 모바일 환경 모두에서 최적의 비율로 보이도록 설계.

## 5. 데이터 태깅 한계 (멀티 알고리즘 지원)

### ❌ 문제 상황
- **증상:** 한 문제에 여러 알고리즘이 포함된 경우 검색 결과가 부정확함.

### ✅ 해결 방법
- **멀티 태그 지원:** 콤마(`,`)로 구분된 다중 알고리즘 태그를 처리할 수 있도록 스크립트 및 챗봇 엔진 로직 수정. (예: 4193번 문제에 'BFS, 그리디' 복합 태그 적용)

---

## 6. 인터랙션 강화 (답변 형식 및 링크 버튼)

### ❌ 문제 상황
- **증상:** 추천 결과가 단순 텍스트 URL로 제공되어 클릭이 불편하고 가독성이 떨어짐.

### ✅ 해결 방법
1.  **답변 형식 카드화:** 텍스트 나열 대신 '제목 - 메타정보 - 버튼' 형태의 카드 레이아웃으로 개편.
2.  **링크 버튼 자동 변환:** 봇 응답 내 마크다운 링크를 감지하여 클릭 가능한 **회색 버튼 형태의 🔗 바로가기 링크**로 변환하는 프론트엔드 로직 추가.
