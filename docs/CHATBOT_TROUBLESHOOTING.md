# 🐱 AI 챗봇 '비사' 트러블슈팅 및 개선 기록

이 문서는 블로그 AI 챗봇 시스템에서 발생한 주요 문제들과 그에 대한 해결 과정을 기록합니다.

## 1. Google Gemini API 지역 제한 (User Location Not Supported)

### ❌ 문제 상황
- **증상:** 챗봇에게 메시지를 보내면 `"냥... User location is not supported for the API use."`라는 에러 메시지가 출력됨.
- **원인:** Cloudflare Worker는 전 세계 데이터 센터에서 실행되는데, 특정 노드(예: 유럽/영국 등)에서 Google AI Studio API를 호출할 때 지역 제한이 걸림.

### ✅ 해결 방법
1.  **API 엔드포인트 변경:** 기존 `v1beta`에서 더 안정적인 `v1` 엔드포인트로 전환.
2.  **인사말 즉시 응답 로직:** "안녕", "ㅎㅇ" 등 단순 인사말은 API를 거치지 않고 Worker 내부에서 즉시 응답하도록 `isGreeting` 로직을 강화하여 API 호출 횟수를 줄이고 에러 가능성을 차단.
3.  **프록시 지원 준비:** 향후 동일 문제 발생 시 우회할 수 있도록 `GEMINI_BASE_URL` 환경 변수 지원 구조 설계.

## 2. 블로그 포스트 URL 경로 불일치

### ❌ 문제 상황
- **증상:** 챗봇이 제공하는 '바로가기' 링크를 클릭하면 404 에러가 발생함.
- **원인:** `problems.json` 생성 스크립트가 파일명 전체(`26-02-11-title.md`)를 URL 슬러그로 사용했으나, 실제 Jekyll 설정은 앞의 날짜(`26-02-11-`)를 제외한 제목 부분만 사용하고 있었음.

### ✅ 해결 방법
- **스크립트 수정 (`generate_problems_json.py`):** URL 생성 함수(`build_post_url`)에서 정규표현식을 사용하여 파일명 앞부분의 날짜 패턴을 제거하도록 로직 수정.
- **데이터 갱신:** 수정된 스크립트를 재실행하여 모든 문제의 링크를 실제 블로그 주소와 100% 일치시킴.

## 3. UI 가독성 및 가시성 문제

### ❌ 문제 상황
- **증상:** 기존 휴대폰 모양의 좁은 UI로 인해 긴 텍스트와 문제 리스트가 한눈에 들어오지 않음.
- **원인:** 고정된 좁은 너비(380px)와 모바일 프레임 디자인의 한계.

### ✅ 해결 방법
1.  **카카오톡 테마 개편:** 가로폭을 최대 600px로 확장하고, 친숙한 카카오톡 스타일(배경색, 말풍선 디자인)을 적용하여 가독성 대폭 향상.
2.  **파일명 정규화:** `music-player.html`을 `chatbot.html`로 변경하여 프로젝트 정체성 확립.
3.  **반응형 레이아웃:** `max-width` 설정을 통해 PC와 모바일 환경 모두에서 최적의 뷰 제공.

## 4. 데이터 태깅 한계 (단일 알고리즘)

### ❌ 문제 상황
- **증상:** 한 문제에 여러 알고리즘이 포함된 경우(예: BFS + 그리디), 검색 결과에 한쪽만 나옴.
- **원인:** 기존 데이터 구조와 스크립트가 문제당 하나의 알고리즘 태그만 처리하도록 설계됨.

### ✅ 해결 방법
- **멀티 태그 지원:** 콤마(`,`)로 구분된 다중 알고리즘 태그를 처리할 수 있도록 스크립트 및 챗봇 엔진 로직 수정.
- **수동 교정:** 4193번 문제 등 주요 문제들에 대해 'BFS, 그리디'와 같이 복합 태그를 적용하여 검색 정확도 향상.

## 5. 인터랙션 불편 (단순 텍스트 URL)

### ❌ 문제 상황
- **증상:** 추천 결과로 나온 링크가 단순 텍스트여서 사용자가 직접 복사해서 주소창에 붙여넣어야 함.

### ✅ 해결 방법
- **링크 버튼 자동 변환:** 봇 응답 내의 마크다운 링크와 URL을 감지하여 클릭 가능한 **회색 버튼 형태의 🔗 바로가기 링크**로 변환하는 기능을 `chatbot.html` 프론트엔드에 추가.
