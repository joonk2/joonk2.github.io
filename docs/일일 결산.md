  🐱 AI 챗봇 '비사' 시스템 통합 개선 보고서

  1. 시스템 안정화 및 에러 완벽 해결
   * Gemini API 지역 제한 극복: Cloudflare Worker의 노드 위치에 따른 접속 제한 문제를 해결하기 위해 API 엔드포인트를
     v1으로 전환하고 안정성을 확보했습니다.
   * 인사말 즉시 응답 시스템: "안녕", "ㅎㅇ" 등 단순 인사는 AI 서버를 거치지 않고 즉시 응답하도록 설계하여, API 에러
     가능성을 원천 차단하고 응답 속도를 0초로 단축했습니다.

  2. UI/UX 및 디자인 전면 개편
   * 카카오톡 스타일 테마: 답답했던 기존 휴대폰 UI를 벗어나, 가로가 넓고 시원한 카카오톡 스타일로 디자인을 전면
     개편했습니다. (사용자 노란색, 봇 흰색 말풍선)
   * 파일명 및 구조 정규화: music-player.html을 chatbot.html로 변경하여 프로젝트 정체성을 확립하고, 사이드바 레이블 및
     팝업 크기(620x750)를 최적화했습니다.
   * 반응형 레이아웃: PC에서는 쾌적하게, 모바일에서는 화면에 꽉 차게 보이도록 유연한 디자인을 적용했습니다.

  3. 문제 추천 로직 및 데이터 고도화
   * 멀티 알고리즘 태그 지원: 한 문제에 여러 알고리즘(예: BFS + 그리디)을 등록할 수 있도록 데이터 생성 스크립트를
     개조했습니다.
   * 데이터 정확도 교정: 4193번(수영대회 결승전) 문제를 'BFS, 그리디'로 올바르게 수정 반영했습니다.
   * 난이도 체계 확립:
       * 쉬움: Programmers Lv 1 / SWEA D1~D2
       * 중간: Programmers Lv 2~3 / SWEA D3~D4
       * 어려움: Programmers Lv 4 / SWEA D5~D6

  4. 인터랙션 및 링크 기능 강화
   * 클릭 가능한 링크 버튼: 봇 응답 내 URL을 감지하여 회색 버튼 형태의 🔗 바로가기 링크로 자동 변환합니다.
   * URL 경로 정밀 수정: 파일명 앞의 날짜(26-02-11-)가 URL에 중복 포함되던 문제를 해결하여, 모든 "바로가기" 버튼이 실제
     블로그 포스트 주소와 100% 일치하도록 수정했습니다.

# 📂 프로젝트 구조 및 명칭 변경 리포트

  1. 파일명 정규화
   * 기존: music-player.html (챗봇 기능이 포함되어 있음에도 음악 기능만 강조됨)
   * 변경: chatbot.html
   * 이유: 메인 기능인 AI 챗봇을 직관적으로 나타내고, 음악 플레이어를 부가 기능으로 포함하는 통합 페이지의 성격을 명확히
     함.

  2. 연동 로직 및 경로 수정 (_includes/head.html)
   * 경로 업데이트: 블로그 내 모든 호출 URL을 music-player.html에서 chatbot.html로 일괄 변경.
   * 팝업 창 크기 최적화:
       * 기존: 400x680 (좁은 휴대폰 스타일)
       * 변경: 620x750 (카카오톡 스타일의 넓은 UI에 최적화)
   * 함수명 변경: openPopupAction 내 내부 변수 및 창 이름을 chatbotPopup 등으로 변경하여 일관성 유지.

  3. 사이드바 UI 레이블 개선 (_includes/sidebar.html)
   * 아이디 변경: music-controls → chatbot-controls
   * 문구 수정: "음악 재생 🎵" 위주의 안내를 "AI 고양이와 대화하기 🐾"로 변경하여 사용자 접근성 향상.
   * 아이콘 및 애니메이션: 챗봇 캐릭터(고양이) 중심의 설명으로 툴팁과 레이블 정리.
